# References

PiCM Factory independently adapts folder-agent and context-engineering ideas into a Pi package. Layouts remain advisory, and the product does not claim one canonical ICM schema.

## Primary methodology source

- [Interpretable Context Methodology: Folder Structure as Agent Architecture](https://arxiv.org/abs/2603.16021) — numbered stages, scoped context, editable intermediate outputs, human review gates, and local scripts/tools for mechanical work.

## External methodology reference

- [`RinDig/icm-architect`](https://github.com/RinDig/icm-architect) — MIT-licensed external skill whose compact cold-agent walk test and file-role inventory concept informed PiCM's independently adapted maintenance and adoption guidance. PiCM uses its own advisory vocabulary and does not adopt rigid naming invariants, Claude-specific defaults, conversion-first migration, or deterministic filesystem-state claims.
